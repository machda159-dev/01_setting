**수신**: 겨울이 (Working Agent)
**발신**: 수리부장 (Product Owner)

**프로젝트**: 포트폴리오 웹사이트 개선/유지보수

---

## 1. 업무 개요 (Task Overview)
현재 포트폴리오 웹사이트(index.html, style.css, script.js 등)의 코드 품질 검수 및 개선 작업을 지시합니다. 겨울이가 이전에 작성한 코드가 요구사항에 맞게 구현되었는지, 웹 표준 및 접근성을 준수하는지, 그리고 반응형 웹 디자인이 적절히 적용되었는지 확인해야 합니다.

## 2. 상세 지시 사항 (Detailed Instructions)

### [ ] 2-1. 코드 리뷰 및 품질 검수 (QA)
- `index.html`, `style.css`, `script.js` 파일 전반에 대한 코드 레벨 리뷰 진행.
- HTML5 시맨틱 태그 사용 여부 점검 및 개선 (현재 section, nav 등은 잘 사용되었는지 재확인).
- 불필요하거나 중복되는 CSS 클래스/규칙 제거 및 최적화.
- JavaScript 에러(콘솔 에러 등) 유무 확인 및 변수/함수명 네이밍 컨벤션 점검.

### [ ] 2-2. 반응형 디자인 (Responsive Web) 테스트 및 오류 수정
- 모바일(뷰포트 320px~768px), 태블릿, 데스크탑 환경에서 UI가 깨지지 않고 정상적으로 렌더링되는지 확인.
- 특히 갤러리 섹션의 이미지 그리드가 모바일 환경에서 적절하게 표시되는지 점검.

### [ ] 2-3. 수정 사항 보고 및 반영
- 검수 과정에서 발견된 문제점 및 개선 피드백 리스트업.
- 수리부장에게 보고 후, 승인을 득한 뒤 코드 수정 반영.

## 3. 완료 조건 (Definition of Done)
- 코드 리뷰 및 QA 리포트 작성 완료.
- 수리부장의 피드백을 반영하여 오류 수정 및 코드 리팩토링 완료.
- 모바일/데스크탑 환경에서 UI 정상 작동 확인.

## 4. 기한 (Deadline)
- **QA 리포트 제출**: 지정된 기한까지.
- **수정 완료**: 승인 후 1일 이내.
